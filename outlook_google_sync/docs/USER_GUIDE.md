# 取り扱い手順書（USER GUIDE）

本書は Outlook→Google カレンダー同期ツールの利用手順を説明します。

---

## はじめに（必ず最初に）

* **読む必要がある場合**: 本ツールで同期・GUI を使う利用者が、**仮想環境を先に作る方針**を理解するとき（初見・運用開始前）。
* **参照を省略してよい場合**: 手順の要否一覧と各章冒頭の `*` だけで足りると判断でき、本方針を既に把握しているとき。

**本ツールを使い始める前に、必ずプロジェクト専用の仮想環境（venv）を構築してください。**

- システム全体にインストールされた Python（グローバル環境）に依存パッケージを入れない運用とする。
- 正規手順では `outlook_google_sync/.venv/` に仮想環境が作成され、`scripts/setup_env.bat` がその作成と依存インストールを行う。
- **`credentials.json` の配置や Google 認証より先に、仮想環境構築を完了させること。**

---

## 手順の要否一覧（いつ必要か／いつ不要か）

* **読む必要がある場合**: どの操作を省略できるか判断したいとき、§1 と §2 の前後関係で迷うとき、初回セットアップのチェックに使うとき。
* **読む必要が薄い場合**: 手順を一通り覚え、迷いがなくなった通常運用の **毎セッション**（必要時にだけ参照）。

次の表は、**その手順を“必ず毎回”やる必要があるわけではない**ものを区別するためのものです。

| 手順・操作 | **必要な場合** | **不要な場合（スキップ可）** |
|------------|----------------|------------------------------|
| **Python 3.10 以上の導入**（§2 手順1） | PC に Python が無い、`python --version` が 3.10 未満 | 既に要件を満たす Python がある |
| **`outlook_google_sync/` の取得**（§2 手順2） | まだフォルダを持っていない（クローン・展開前） | リポジトリ／配布物を既に配置済み |
| **`setup_env.bat`（仮想環境構築）** | **初回**、`.venv` を消した、`requirements.txt` 変更後に依存を入れ直したい、モジュール不足で起動しない | **`.venv` が存在し** 依存が入った状態で問題なく動いている **通常の毎回の作業** |
| **`credentials.json` の配置**（§3） | **Google カレンダー API を使う**（本ツールの本来の用途） | ソース閲覧のみ等、Google 連携を一切しない場合（稀） |
| **Google 認証（GUI）**（§4） | 初回、トークン失効、`token.json` が無い／無効 | 直前まで認証済みでトークンが有効 |
| **`run_gui.bat`（GUI 起動）** | アプリで同期・設定・テストをする **その都度** | 作業しないセッション |
| **接続テスト**（§8） | 環境構築直後、入力方法や PC を変えた、エラーで原因切り分けしたい | 繰り返し利用で既に疎通に問題がない通常運用のみ |
| **通常プレビュー** | 本番書込前に内容を確認したい（推奨） | 自己責任で省略可能 |
| **強制フル同期 / フルプレビュー** | 設定の大変更後、件数異常・不整合・重複疑い | 日々の更新は **通常同期／通常プレビュー** だけで足りることが多い |
| **重複修復**（§7） | 同一キー等の重複が疑われる・発生している | 重複が無く、修復の必要がない |
| **手動で `.venv` を作る**（§2.1） | `setup_env.bat` が使えない、自前で venv を管理したい | `setup_env.bat` で問題なく構築できる |
| **`Activate.ps1` で venv 有効化** | **`python` / `pip` をターミナルから直接叩く** とき | **`run_gui.bat` や `.venv\Scripts\python.exe` だけ使う** なら必須ではない |
| **テスト（pytest）**（§13） | 開発・検証、不具合再現後の確認 | エンドユーザーがツール利用のみの場合 |

**§1 と §2 の関係**

- **§1 の表**は、`setup_env.bat` と `run_gui.bat` の **最小まとめ**です。
- **§2 の手順3**（`setup_env.bat`）は、§1 の「**初回のみ**」の行と **同じ作業**です。**両方を二重に実行する必要はありません**（再実行しても害は通常ありませんが、初回済みなら §2 の3は“もう終わっている”）。
- **§2 の手順1・2・4**は §1 の表には書いていませんが、初回には **別途必要** なことがあります（Python・フォルダ・credentials）。
- **経験者**: §2 はチェックリストとして「まだやっていない項目」だけ確認すれば十分です。

* **実施が必要な場合**: **初回**は `setup_env.bat`、**作業するたび**に GUI を使うならそのセッションで `run_gui.bat`（または既存の起動手順）。

* **参照が必要な場合**: Cursor／PowerShell で `.bat` や `python` を実行するとき、実行場所で失敗するとき。
* **参照が不要な場合**: すでに「`outlook_google_sync` でターミナル実行」と決め打ちできているとき。

本書で `*.bat` やコマンド実行を指示する場合、原則は次のとおり。

1. **推奨**: Cursor のターミナル（PowerShell）で実行する  
   - 事前に `outlook_google_sync/` をカレントディレクトリにする
2. **代替**: Windows PowerShell（単体）で実行してもよい
3. **補助**: エクスプローラから `.bat` をダブルクリック実行してもよい（失敗時にログ確認しづらいため、推奨はターミナル実行）

### 1.2 カレントディレクトリ変更コマンド（必須）

* **実行が必要な場合**: ターミナルを開いた直後で、カレントが `outlook_google_sync/` でないとき。
* **実行が不要な場合**: 既にそのフォルダで作業しているとき（`cd` の繰り返しは不要）。

本書で `outlook_google_sync/` への移動が必要な手順では、次のコマンドを **必ず最初に** 実行する。

```powershell

```

> 注意: `cd scripts\setup_env.bat` のように **ファイルへ `cd` はできない**。  
> `.bat` は `cd` 後に `.\scripts\setup_env.bat` / `.\scripts\run_gui.bat` のように実行する。

### 1.3 メインウィンドウの終了・操作中のボタン

- **メインウィンドウのタイトルバー ×**: **確認なしでプロセスを即終了**する（未保存の入力は残らない）。設定は通常どおり JSON へ保存済みのもののみ残る。
- **同期・プレビュー・接続テスト等の処理中**: 主要ボタン・日付・ICS「参照」は **無効（グレー）** になり、誤操作しにくい。**キャンセル** だけ有効。
- **設定の「Google 認証」「カレンダー一覧を取得」**: 通信中は **両方とも一時的に無効** になり、**保存・キャンセル・既定値に戻す・プロファイルのエクスポート／インポート** も無効（誤保存防止）。完了後に再び操作できる。ブラウザでの許可中も GUI は応答し続け、メインの × で抜けられる（認証は途中で破棄される場合あり）。
- **メインの処理中**: **ICS ファイルパスの入力欄** と **参照**、**開始日／終了日** も無効。
- **極端に応答しない** ときは、上記 × に加え **タスクマネージャーで終了** も従来どおり可能。

## 3. credentials.json 配置

* **実施（準備）が必要な場合**: **Google カレンダー API を使う**とき（本ツールの通常利用）。初回、OAuth クライアントを作り直したとき、別PCへ移したとき。
* **実施が不要な場合**: Google 連携を一切しない限定的な利用（ソース閲覧のみ等。稀）。

### 3.1 何を配置するか（前提）

- `credentials.json` は、Google Cloud で作成した **OAuth クライアントID（デスクトップアプリ）** の JSON ファイル。
- 本ツールは実行時に `~/.outlook_google_sync/credentials.json` を参照する。
- `credentials.json` と `token.json` は別物:
  - `credentials.json`: アプリ側の OAuth 設定（初期設定ファイル）
  - `token.json`: 認証後に生成されるアクセストークン情報

### 3.2 Google Cloud Console 側の作業

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成（または既存選択）
2. 「API とサービス」→「ライブラリ」で **Google Calendar API を有効化**
3. 「API とサービス」→「OAuth 同意画面」を設定（テスト運用ならテストユーザーに自分を追加）
4. 「API とサービス」→「認証情報」→「認証情報を作成」→「OAuth クライアント ID」
5. アプリケーションの種類は **デスクトップアプリケーション** を選択
6. 作成後に JSON をダウンロード

### 3.3 ローカルPCへの配置手順（Windows）

1. エクスプローラのアドレスバーに `%USERPROFILE%` を入力して開く
2. `.outlook_google_sync` フォルダを作成（既にあればそのまま）
3. ダウンロードした JSON を `credentials.json` にリネーム
4. 次の場所に配置:
   - `%USERPROFILE%\.outlook_google_sync\credentials.json`
5. **開発フォルダ (`outlook_google_sync/`) には配置しないこと**

### 3.4 配置確認（迷ったとき）

PowerShell で次を実行し、`True` が返ることを確認:

```powershell
Test-Path "$env:USERPROFILE\.outlook_google_sync\credentials.json"
```

任意で中身の形式を確認（`installed` があれば通常はデスクトップアプリ形式）。**仮想環境を使っている場合**は、プロジェクト直下で次のように **`.venv` の Python** を指定する（グローバル `python` でも可）:

```powershell
cd D:\workspace\amatsuka_argo_5_obsidan_chatgpt_cursor_genspark\outlook_google_sync
.\.venv\Scripts\python.exe -c "import json, pathlib; p=pathlib.Path.home()/'.outlook_google_sync'/'credentials.json'; d=json.loads(p.read_text(encoding='utf-8')); print('installed' in d)"
```

### 3.5 よくある誤り

- `client_secret_xxx.json` の名前のまま置いてしまう（`credentials.json` に統一）
- API 有効化前に認証を進める（Calendar API を先に有効化）
- Web アプリ種別で作成してしまう（本ツールはデスクトップアプリ前提）
- リポジトリ配下に置いてしまう（誤コミット防止のため実行時データフォルダに置く）

## 4. Google 認証

* **実施が必要な場合**: **初回**のGoogle連携、`token.json` が無い／**トークン失効**で認証エラーが出るとき、別アカウントに切り替えるとき。
* **実施が不要な場合**: 有効な `token.json` があり、Google操作で認証エラーが出ていないとき。

**前提**: `credentials.json` は §3 のとおり **実行時データフォルダ**に配置済みであること。

### 4.1 手順（GUI）

1. `.\scripts\run_gui.bat` で GUI を起動する
2. 設定ウィンドウの **「Google」** タブを開く
3. **「Google認証」** を押す（またはブラウザの許可画面が自動で開く）
4. 承認すると **`token.json`** が作成・更新される

- **トークン保存先**: `%USERPROFILE%\.outlook_google_sync\token.json`
- **OAuth スコープ**: `https://www.googleapis.com/auth/calendar`

**別の Google アカウントに切り替える場合**:

- ブラウザで既存セッションが残っていると、意図しないアカウントで進むことがある。**シークレット／プライベートウィンドウ**で認証し直すか、Google から一度ログアウトしてからやり直す。
- 切り替え後は **`token.json` を削除**し、§4.1 から初回認証をやり直すと確実。

### 4.2 ブラウザで「アクセスをブロック」「403: access_denied」になる場合

OAuth 同意画面の **公開ステータスが「テスト中（Testing）」** のとき、Google は **テストユーザーに登録されたアカウントだけ**が認証できます。登録されていないアカウントで許可すると、次のような表示になることがあります。

- 「**アクセスをブロック**: … は Google の審査プロセスを完了していません」
- 「**エラー 403: access_denied**」

**解決手順（個人利用・自分用に認証する場合）**

1. [Google Cloud Console](https://console.cloud.google.com/) で、`credentials.json` を発行した **同じプロジェクト** を開く
2. **「API とサービス」→「OAuth 同意画面」** を開く
3. ページ下の **「テストユーザー」** で **「ユーザーを追加」**
4. **認証に使う Gmail アドレス**（ブラウザに表示されているアカウントと完全一致）を追加して保存
5. 数分待ってから、本ツールから **Google 認証を再度実行**する

**補足**

- 家族・別PCで別アカウントを使う場合も、**そのアカウントごと**にテストユーザーを追加する。
- 同意画面を **「本番（In production）」** にすると誰でも許可画面に進める場合があるが、**カレンダー等のスコープ**によっては Google の **アプリ検証**が必要になり、自分用なら **テストユーザー追加の方が簡単**なことが多い。

## 5. 入力方法の使い分け

* **参照が必要な場合**: **COM／手動ICS／マクロ**のどれでOutlook予定を読むか決めるとき、**入力方法を切り替える**前後。
* **参照が不要な場合**: 運用を固定し、しばらく入力方法を変えないとき（内容を既に把握していれば毎回読む必要はない）。

| 入力方法 | 条件 | 切替時の注意 |
|----------|------|------------|
| **COM** | Classic Outlook + Windows + pywin32 | 企業GPOで制限される場合あり |
| **手動 ICS** | Outlookからエクスポートした `.ics` | ファイルを都度選択 |
| **マクロ連携** | VBA マクロで `.ics` 出力 | `samples/macro/export_calendar.bas` を参照 |

**重要**: COM ↔ ICS を切り替えると `sync_key` が異なるため重複の可能性あり。切替時に確認ダイアログが表示される。

## 6. 同期実行手順

* **実施が必要な場合**: 指定期間の予定を **Google に反映したい**とき（本ツールの主用途）。
* **実施が不要な場合**: その間に同期作業をしないとき（GUI を開いても同期ボタンを押さないなら当章の操作は不要）。

### 通常同期（差分）

* **選ぶべき場合**: 日々の更新・差分だけ反映したい **通常運用**（既定の同期モード）。
* **不要になる場合**: 常に全件からやり直したいときは「強制フル同期」を検討（下記）。
1. `.\scripts\run_gui.bat` を実行して GUI を起動（Cursor ターミナル推奨）
2. 開始日・終了日を指定
3. 「通常同期」ボタンを押す
4. 差分のある予定のみ Google に書込

```powershell
cd D:\workspace\amatsuka_argo_5_obsidan_chatgpt_cursor_genspark\outlook_google_sync
.\scripts\run_gui.bat
```

### 強制フル同期

* **実施が必要な場合**: **設定やフィルタを大きく変えた後**、件数異常・**不整合・重複疑い**、差分メタデータが信じられないとき。
* **実施が不要な場合**: 日々の予定変更だけを反映したい **だけ**のときは、通常同期で足りることが多い。

1. GUI 起動済みであることを確認
2. 開始日・終了日を指定
3. 「強制フル同期」ボタンを押す
4. 対象期間の全予定を再評価して Google に書込

**強制フル同期の使いどころ**:
- 大きな設定変更後の整合再構築
- 件数異常・重複疑いの修復
- 差分メタデータの不整合が疑われるとき

### プレビュー同期

* **実施が必要な場合**: **本番書込前**に取り込み内容を確認したいとき（推奨）。**フルプレビュー**は強制フル相当の見積りが欲しいとき。
* **実施が不要な場合**: 内容確認を省略してよいと判断するとき（自己責任でスキップ可）。

1. 「通常プレビュー」または「フルプレビュー」ボタン
2. 想定アクション一覧（新規/更新/削除/スキップ/マージ/重複）を確認
3. 個別除外や一括承認が可能
4. 「本番同期実行」で書込実行

**注意**: プレビュー後5分以上経過すると「再プレビュー確認」ダイアログが表示される。

## 7. 重複修復ツール

* **実施が必要な場合**: 同一 `sync_key` など **重複が疑われる・発生した**とき（COM↔ICS切替後など）。
* **実施が不要な場合**: 重複がなく、カレンダー上で問題がないとき。

1. GUI 起動済みであることを確認（未起動なら `.\scripts\run_gui.bat` を実行）
2. メインウィンドウの「重複修復」ボタン
3. 同一 `sync_key` の重複グループが一覧表示される
4. 残すイベントを選択 → 他を削除
5. 「古い方を残す（一括）」で自動選択も可能

## 8. 接続テストの意味と限界

* **実施が必要な場合**: **環境構築直後**、**PCや入力方法を変えた後**、エラー時の**切り分け**、不安なときの疎通確認。
* **実施が不要な場合**: 安定運用で追加入力もなく、追加の疎通確認が不要なとき（毎回必須ではない）。

- **疎通と最小読取の確認** であり、同期の全操作の成功は **保証しません**
- COM: 既定カレンダーを開き最低1件読取。0件は警告だが成功扱い
- ICS: 11項目の詳細チェック（ファイル存在・構造・パース・UID・期間内件数等）
- Google: 2段構成
  1. トークン有効性 + calendarList.list（基本接続）
  2. 対象カレンダー選択済みなら calendarId 存在確認
  - 対象カレンダー未選択時は「未実施」メッセージ（失敗扱いにしない）

## 9. ログの見方

* **参照が必要な場合**: UI の短文では足りず**詳細**が欲しいとき、**障害調査**、ログ詳細度（standard / detailed）を変えたいとき。
* **参照が不要な場合**: GUI のメッセージだけで十分なとき（ファイルログを開かなくてよい）。

- **保存先**: `%USERPROFILE%\.outlook_google_sync\logs\`
- **ファイル名**: `sync_YYYY-MM-DD.log`（日付ローテーション）
- **レベル**:
  - `standard`（既定）: 成功/失敗、件数、要約
  - `detailed`: 判定理由、API応答、例外詳細
- レベルは設定ウィンドウ「ログ/通知」タブで切替
- ログ確認は、エクスプローラまたは PowerShell のどちらでも可

## 10. 非公開予定・情報持ち出しリスク

* **読む必要がある場合**: **非公開予定を含める設定**にする前／した後、**情報持ち出し・組織ポリシー**を確認したいとき。
* **読む必要が薄い場合**: 非公開を含めず、当該リスクが表面化しない運用のとき（それでも組織ルールの順守は別途必要）。

- **非公開予定を含める = ON** の場合、タイトル・本文・場所もすべて Google に送信される
- Google 側では `visibility=private` が設定されるが、**情報は Google アカウントに保存される**
- 組織のセキュリティポリシーに従って利用すること
- `full` で同期後に `title_only` に切り替えても、既に送信された `description` / `location` は Google 上に **残る**

## 11. よくある失敗例と対処

* **参照が必要な場合**: **エラーや不具合が出たとき**、起動前に**想定トラブル**を把握したいとき。
* **参照が不要な場合**: 問題がなく、トラブルシュートに回さないとき。

| 症状 | 対処 |
|------|------|
| `ModuleNotFoundError` や GUI が起動しない | **先に** `.\scripts\setup_env.bat` で `.venv` を作成したか確認。未作成なら作成する |
| `run_gui.bat` が認識されない（PowerShell） | カレントが `outlook_google_sync/` か確認し、`run_gui.bat` ではなく **`.\scripts\run_gui.bat`** と実行する |
| COM 接続テスト失敗 | `ics_manual` / `ics_macro` に切替 |
| 認証失敗・トークン期限切れ | `credentials.json` の配置確認、設定から再認証 |
| 「アクセスをブロック」／**403: access_denied** | OAuth 同意画面がテスト中のとき。**Cloud Console → OAuth 同意画面 → テストユーザー** に、認証に使う Gmail を追加（§4.2） |
| メインが固まったように見える | 処理中は **キャンセル** か **メイン ×（即終了）**。設定の Google 操作はバックグラウンド化済み（§1.3） |
| config.json migration 非互換 | `config.backup.v*.json` がバックアップ済み。再設定が必要 |
| 重複イベント発生 | 「重複修復」ツールで対処 |
| 予定が 0 件 | 期間指定を確認。接続テストでは警告だが失敗ではない |
| ICS パースエラー | 接続テストで詳細チェック。文字コードやTZ情報を確認 |
| Google API `Missing time zone definition for start/end time`（400） | 本ツールは **同期先カレンダーの `timeZone`（IANA）を API で取得**し、時刻付きイベントの `start`/`end` に付与して送信する。対象カレンダーが正しく選ばれているか確認する |

補足:
- `.bat` が起動しない場合は、**エクスプローラのダブルクリックではなく** Cursor ターミナル（PowerShell）で `.\scripts\xxx.bat` として実行する
- PowerShell の実行ポリシーで止まる場合は、管理者へ相談する（自己判断で緩和しない）

## 12. 設定ウィンドウ

* **開く必要がある場合**: **フィルタ・衝突ポリシー・詳細度・カレンダーID・通知**などを変更するとき、**エクスポート／インポート**するとき。
* **開かなくてよい場合**: 既定・現状の設定のまま同期だけ繰り返すとき（変更が無ければ不要）。

メインウィンドウの「設定」ボタンから開く。以下のタブで構成:

| タブ | 内容 |
|------|------|
| 基本 | プロファイル選択 |
| 同期 | 入力方法、詳細度、衝突ポリシー、マージ設定 |
| フィルタ | 終日・Free・Tentative・Private除外、カテゴリ、キーワード |
| Google | 対象カレンダー選択、認証 |
| ログ/通知 | ログ詳細度、通知ON/OFF |
| 高度な設定 | 将来拡張用 |

保存・キャンセル・既定値リセットボタンあり。ウィンドウ位置・サイズは保存される。

## 13. 実行場所の早見表

* **参照が不要な場合**: 実行場所に慣れ、迷わないとき。

| 通常同期/フル同期/プレビュー | アプリ GUI | ボタン操作で実行 |
