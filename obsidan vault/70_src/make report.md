    + 新規作成のレポートは以下の基準に則ってファイル名とYAMLを更新して。
    + 元のファイルのファイル名は出力レポートのファイル名と同じに変更して。
    + レポートの冒頭（YAMLの後）に概要と目次を付けて

+ 【ファイル名の命名基準】
    ファイル名はYYYYMMDD_A_B_C
    + YYYYMMDDは作成日次
    + A：作者（Cursor）
    + B：資料と関連するテーマやセクター
    + C：中身の概要

+ 【YAMLの形式】
    + tags:レポートの内容にから考えて記入。20_Literature、30_Permanentのファイルを参考にすること。related_sectors, related_news, related_stocksの項目は全てここにも記載すること。
    + related_sectors:関連テーマや関連セクターを考えて記入。20_Literature、30_Permanentのファイルを参考にすること
    + related_news:関連ニュースを考えて記入。20_Literature、30_Permanentのファイルを参考にすること
    + related_stocks:関連銘柄を考えて銘柄名（銘柄コード）で記入。20_Literature、30_Permanentのファイルを参考にすること
    + source: ソースへのリンクを記入。
    + author: 作者を記入（youtubeなら投稿者、chatgptならchatgpt, 不明なら作者不明）
    + created: YYYY-MM-DD
    + modified: YYYY-MM-DD
